Claude Code — The CLI Version of What You Just Saw

The app is fun. The CLI is a different animal. This is a walk-through of installing Claude Code on your Mac and a tour of how I have mine set up — so you can see what it feels like when Claude lives in your terminal and can actually do things.

Pro tip: For a better voice, select all then use iOS Speak Selection (Settings > Accessibility > Spoken Content).

Hey Sara — this is a little tour. You just got clawed in the app, which is a nice on-ramp, but the real unlock is the CLI. That's the version where Claude can read your files, run commands, push to GitHub, and stay in a project with you for hours. This doc walks you through installing it and shows you how I have mine wired up. At the bottom there's a link to a repo I put together for you — you clone it, and it onboards you the rest of the way.

What "the CLI" actually means

Claude Code is a command-line tool. You open your terminal, type claude, and you're talking to Claude the same way you talked to it in the app — but now it can edit files on your computer, run shell commands, commit to git, and use a growing library of skills and MCP servers (more on those in a minute).

The app is a chat window. The CLI is an agent. Big difference.

Step 1 — Install the prerequisites

If you've never used a terminal before, open Terminal.app (Spotlight: Cmd+Space, type "Terminal"). Paste these in one at a time.

Homebrew (the package manager for Mac)

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

It'll ask for your Mac password. That's fine — it's the standard installer.

Node.js (Claude Code is a Node package)

brew install node

Git and the GitHub CLI

brew install git gh

Then sign in to GitHub so you can push repos later:

gh auth login

Step 2 — Install Claude Code

npm install -g @anthropic-ai/claude-code

That's it. To start it, go to any folder you want to work in and type:

cd ~/Desktop
claude

First run, it'll open a browser tab for you to log in with your Anthropic account. After that, you're in.

You're talking to Claude now. Anything you type in the terminal is a prompt. It reads files, it edits files, it runs commands. The whole computer is its workspace.

Step 3 — The four permission modes

This is the most important concept to grasp early. Press Shift+Tab to cycle through the first three:

Normal

Claude asks before doing anything that touches your computer. Great while you're learning.

Auto-accept

Claude edits files and runs safe commands without asking. Still shows you what it's doing.

Plan mode

Read-only. Claude can look and think but can't change anything. Good for "explain this codebase."

YOLO (dangerous)

claude --dangerously-skip-permissions — no asking at all. Only in folders where you can afford a mistake.

Start in Normal mode. After a week or two you'll hop into Auto-accept for most work. I basically live in Auto-accept with a /yolo skill (more on those in a minute).

Step 4 — The things that make it feel like yours

Out of the box, Claude Code is powerful. What makes it feel personal is three things: CLAUDE.md, memory, and skills. These are the reasons my Claude feels like my assistant and not just a generic AI.

CLAUDE.md — your standing instructions

A file called CLAUDE.md at the root of a project gets loaded into every Claude Code session in that folder. Mine has my name, my job (videographer in Cookeville TN), how I want it to talk to me (terse, no emoji, don't ask unnecessary questions), and rules about my workflow. Think of it as the system prompt for that folder.

A CLAUDE.md at ~/.claude/CLAUDE.md applies globally — to every folder on your machine.

Memory — persistence across conversations

Claude can save little markdown files to a memory folder. When you start a new conversation, it can recall who you are, what you care about, and feedback you've given before. No more re-explaining yourself every morning. I have about 50 memory files — all of them under a hundred words.

Skills — reusable capabilities

Skills are markdown instructions stored at ~/.claude/skills/. You invoke them with a slash command. A skill called /commit might tell Claude exactly how to write a git commit message the way you like. A skill called /yolo flips it into auto-accept mode. Skills compose — you can give Claude a whole behavior with one word.

Step 5 — MCP servers (the superpowers)

MCP (Model Context Protocol) lets Claude plug into outside tools. You add a little config to your settings.json and suddenly Claude can read your Gmail, drive a browser, look at a Postgres database, or query live documentation. Here's what I actually use day-to-day:

Matt's MCP Servers
  • sequential-thinking — lets Claude think step-by-step on hard problems
  • context7 — pulls live docs for any library or framework so answers aren't stale
  • chrome-devtools — drives a headless Chrome for scraping, testing, or checking a page
  • knowledge-graph — a small persistent memory DB for longer-term facts
  • Gmail personal + Calendar — Claude reads my inbox and schedules events directly
  • Gmail business — same, separate account for my video business

You don't need all of these to start. I'd start with just sequential-thinking and context7 — they're free, they work out of the box, and they make Claude noticeably smarter.

How my setup is actually wired

Not prescriptive — just so you can see what a real working setup looks like after a few months of iteration.

  • Global CLAUDE.md at ~/.claude/CLAUDE.md with my voice-TTS toggles (my computer literally talks back to me when Claude finishes responding)
  • Per-project CLAUDE.md in every folder that matters — tells Claude the rules for that specific project
  • Memory folder with 50-ish files covering who I am, feedback I've given, ongoing projects, and external system references
  • Cortex — a separate Obsidian vault at ~/Cortex that's my long-form knowledge base. Claude reads it, writes to it, keeps it tidy
  • Skills — around 20 active slash commands, plus an archive of ~30 more I toggle on/off depending on the project
  • RandomPhoneBuilds — a GitHub Pages site where I publish research docs like the one you're reading right now. I just say "put this on RPB" and Claude pushes a styled HTML page live in about a minute
  • Neo — a little agent that watches my iMessage inbox and routes requests to Claude when I'm away from my computer

You don't need Neo or Cortex or any of that to be useful. They came one at a time. The only things that matter day one are Claude Code installed, a CLAUDE.md file, and the permission modes.

The three slash commands I'd install first

These live in ~/.claude/commands/ (or ~/.claude/skills/ depending on your version). Each is a tiny markdown file.

/yolo

Auto-approves file ops, git, and safe dev tools. Still asks before rm -rf or sudo. This is my default mode.

/safe

The opposite. Asks permission for everything. Use when you're doing something you don't fully trust.

/commit

Looks at your staged and unstaged changes, writes a clean commit message that explains the why, and commits it. Never stops to ask trivia.

All three are bundled in the onboarding repo I made for you (linked below). You don't have to write them from scratch.

Keyboard shortcuts worth memorizing

  • Enter — send your message
  • Shift+Enter — new line without sending (for multi-line prompts)
  • Shift+Tab — cycle permission modes (Normal → Auto-Accept → Plan)
  • Escape — clear the current input
  • Escape, Escape — (with empty input) opens the rollback menu — lets you undo your conversation to any earlier point
  • / — open the slash-command menu
  • claude --resume — reopen your last session after quitting

Tips from the dirt

  • Be specific. "Build a website" is too vague. "Make an index.html with Tailwind and a hero, nav, and footer" gets better output fast.
  • Let it read before you ask it to write. "Read these three files, then tell me what you think" is almost always the right opener.
  • Course-correct bluntly. If it's going the wrong way, just say "stop, that's not what I meant, I want X." You're not hurting its feelings.
  • Use Plan Mode to learn. It'll tell you what it would do without actually doing anything. Great for understanding a codebase.
  • Commit often. Claude is a fast editor. A bad edit session is recoverable if you've been committing. Use /commit every 20 minutes.
