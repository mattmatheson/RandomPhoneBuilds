The Technology

The tools professionals use.

Three pieces of software. That's all you need to build a professional-grade audio plugin and sell it to every Logic Pro, Ableton, and Pro Tools user on earth.

01
JUCE
The Framework

The industry-standard C++ framework for audio software. Used by Waves, Native Instruments, iZotope — basically everyone. Write once, export to AU, VST3, and AAX simultaneously. Has everything built in: knobs, sliders, meters, DSP math, and a full plugin hosting environment for testing.

juce.com
02
C++
The Language

Real-time audio demands real-time performance. C++ is the only language that gives you the speed to process audio buffers at 44,100+ samples per second without a single dropout. JUCE handles most of the hard parts — you just write the DSP math that makes your plugin sound unique.

Industry standard
03
Xcode
The Build System

Apple's free IDE is required to compile Audio Units for Logic Pro. JUCE's Projucer tool generates the Xcode project automatically — you mostly just use Xcode to build, debug, and sign your plugin. Apple notarization is required for macOS Ventura+ users to install your plugin without security warnings.

Free on Mac App Store

Core DSP — Circular Buffer Delay (C++ / JUCE)
// Inside processBlock() — called 44,100+ times per second
void processBlock(juce::AudioBuffer<float>& buffer, juce::MidiBuffer&)
{
    auto totalNumSamples = buffer.getNumSamples();

    for (int sample = 0; sample < totalNumSamples; ++sample)
    {
        float inputSample = buffer.getSample(0, sample);

        // Read delayed sample from circular buffer
        float delayedSample = delayBuffer[readPos];

        // Write input + feedback into buffer
        delayBuffer[writePos] = inputSample + (delayedSample * feedbackAmount);

        // Mix dry and wet signals
        float output = (inputSample * dryMix) + (delayedSample * wetMix);
        buffer.setSample(0, sample, output);

        // Advance buffer positions (wrap around)
        writePos = (writePos + 1) % bufferSize;
        readPos  = (readPos  + 1) % bufferSize;
    }
}

The Plan

Zero to first sale in 8 weeks.

A realistic, week-by-week path from first download to plugin on sale. No shortcuts — but no wasted steps either.

1–2
Weeks 1–2
Environment Setup & JUCE Basics
Download Xcode, download JUCE, generate your first plugin project with Projucer. Build a simple gain plugin that compiles and loads in Logic Pro. Understand the audio buffer model and the processBlock() loop.
Xcode JUCE Projucer AU Plugin processBlock()
3–4
Weeks 3–4
Delay DSP Implementation
Build the circular buffer, implement delay time, feedback, and wet/dry mix. Add BPM sync. Test with actual audio in Logic Pro. This is the core of your product — get it sounding right before touching the GUI.
Circular Buffer Feedback BPM Sync DSP Math
5–6
Weeks 5–6
GUI Design & Differentiation
Design your plugin's look in JUCE. Custom knobs, a waveform visualizer, maybe an X/Y pad. Add what makes your delay unique — a filter in the feedback path, stereo ping-pong, or tape flutter via LFO modulation. This is your brand.
JUCE Graphics Custom Knobs LFO UI Design
7–8
Weeks 7–8
Ship It
Apple notarize your binary (required since macOS Ventura). Set up Gumroad for instant indie sales — then apply to Plugin Boutique for wider reach. Write a product page, record a demo video, price it at $29–$49. You're in business.
