Setting Up Claude Code on Windows

The easy way. WSL2 + one script. Your Mac setup, cloned to Windows.

Pro tip: For a better voice, select all then use iOS Speak Selection (Settings > Accessibility > Spoken Content).

This guide gets Claude Code running on your Windows machine through WSL2 (Windows Subsystem for Linux). Everything transfers from your Mac setup -- memory, skills, Firecrawl, configs. One script, five minutes.

What You're Getting

Your full Mac Claude Code environment, rebuilt on Windows. Same memory files, same MCP servers, same workflows. The setup script handles all the tedious parts.

What won't work

Mac only

iMessage integration and the Neo agent system are macOS-exclusive. They rely on the Messages database and AppleScript, which don't exist on Windows. Everything else works.

Prerequisites

  • Windows 10 (version 2004+) or Windows 11
  • Internet connection
  • Your Anthropic account credentials

That's it. The setup script installs everything else.

Step 1

Install WSL2

Open PowerShell as Administrator. Right-click the Start button and pick "Terminal (Admin)" or search for PowerShell and choose "Run as administrator."

Run this single command:

wsl --install

This installs WSL2 with Ubuntu. When it finishes, restart your computer.

After restarting, open Ubuntu from the Start menu. It will finish setting up and ask you to create a username and password. Pick something simple -- you'll type this password occasionally for sudo commands.

Once you see the Ubuntu terminal prompt, you're in Linux. Run a quick update:

sudo apt update && sudo apt upgrade -y
Step 2

Run the Setup Script

Download and run the setup script. It installs Node.js, Claude Code, Firecrawl, and copies over all your configs.

curl -fsSL https://raw.githubusercontent.com/mattmatheson/RandomPhoneBuilds/main/claude-windows-setup/setup.sh -o setup.sh

Review it if you want (always a good idea), then run it:

bash setup.sh

What it installs

  • nvm + Node.js LTS -- Claude Code runs on Node
  • Claude Code CLI -- via npm
  • Firecrawl -- web scraping for research
  • Your configs -- copies CLAUDE.md, memory files, MCP settings from the backup tarball
Done right

The script is idempotent -- you can run it again safely if something goes wrong.

Step 3

Authenticate

Launch Claude Code for the first time:

claude

It'll open a browser window to log in with your Anthropic account. Sign in and approve the connection.

If you need to use an API key instead, you can set it as an environment variable:

export ANTHROPIC_API_KEY="your-key-here"

Add it to your ~/.bashrc if you want it to persist across sessions.

Step 4

Reconnect MCP Servers

Your MCP server configs were copied over, but the OAuth tokens don't transfer. Each service will prompt you to re-authorize on first use. It's just clicking "Allow" in a browser popup.

  • Google Calendar -- prompts OAuth on first calendar command
  • Gmail -- prompts OAuth on first email command
  • Notion -- prompts OAuth on first Notion command
  • Canva -- prompts OAuth on first Canva command
Heads up

If the browser popup doesn't open automatically, check the terminal output for a URL to paste into your browser manually.

Step 5 (Optional)

Sync Setup

Want your Mac and Windows to stay in sync? Both machines can share memory and context through git.

Clone the Cortex vault

If you use the Obsidian Cortex vault, clone it on Windows too:

git clone https://github.com/mattmatheson/cortex-vault.git ~/Cortex

Use the sync script

The setup script installs a sync.sh helper. Run it to pull latest memory and configs:

bash ~/sync.sh

This pulls the latest from your shared repos so both machines have the same context.

Step 6: Obsidian + Cortex Vault (Your Second Brain)

Your Obsidian Cortex vault is already git-synced on your Mac. Setting it up on Windows means both Claudes share the same knowledge base.

Install Obsidian (Windows — not WSL)

Download Obsidian from obsidian.md and install it as a regular Windows app.

Clone the Cortex vault

In WSL, clone your vault where Windows can also access it:

git clone git@github.com:mattmatheson/cortex-vault.git /mnt/c/Users/Matt/Cortex

Note: /mnt/c/Users/Matt/ — change "Matt" to whatever your Windows username is. We put it on the C: drive so both WSL and Windows Obsidian can access it.

Open in Obsidian

Open Obsidian on Windows → "Open folder as vault" → navigate to C:\Users\Matt\Cortex

Install GitSync plugin (optional)

Same plugin you use on your phone. Settings → Community plugins → search "Git" → install "Obsidian Git". Configure it to auto-pull/push every 5 minutes. This keeps your vault in sync across Mac, Windows, and iPhone.

Symlink for Claude Code (so it finds Cortex at ~/Cortex)

ln -s /mnt/c/Users/Matt/Cortex ~/Cortex

Now Claude Code on Windows can read ~/Cortex/ just like it does on Mac.

How sync works across machines: Both machines push/pull to the same GitHub repo. Mac Claude writes a session note → git pushes → Windows Claude pulls it next conversation. Same brain, different bodies.

Troubleshooting

WSL won't install

Make sure virtualization is enabled in your BIOS. Search "Turn Windows features on or off" and ensure "Virtual Machine Platform" and "Windows Subsystem for Linux" are both checked. Restart after enabling.

Node version issues

Claude Code needs Node 18 or higher. Check your version:

node --version

If it's too old, update through nvm:

nvm install --lts && nvm use --lts

Permission errors

If you get EACCES errors installing global npm packages, fix npm's default directory:

mkdir -p ~/.npm-global npm config set prefix '~/.npm-global' echo 'export PATH="$HOME/.npm-global/bin:$PATH"' >> ~/.bashrc source ~/.bashrc

WSL networking issues

If you can't reach the internet from WSL, try resetting the network stack:

sudo rm /etc/resolv.conf sudo bash -c 'echo "nameserver 8.8.8.8" > /etc/resolv.conf' sudo chattr +i /etc/resolv.conf
