Sony HVR-MRC1 + Panasonic DVX100

How to use the Sony HVR-MRC1 Memory Recording Unit to record DV from a Panasonic AG-DVX100 to CompactFlash via FireWire — complete troubleshooting guide.

Pro tip: For a better voice, tap select all below then use iOS Speak Selection (Settings > Accessibility > Spoken Content > Speak Selection).

You have a Sony HVR-MRC1 Memory Recording Unit and a Panasonic AG-DVX100. The DVX100 connects fine to your MacBook via FireWire (QuickTime capture works), but when you plug the same cable into the MRC1, it won't record. This guide covers exactly what's going on and how to fix it.

The good news: this combination has been confirmed working by multiple users on DVXuser and DVinfo forums. The bad news: it requires specific settings on both devices, and the MRC1 is not exactly intuitive when used outside of its native Sony ecosystem.

The Short Answer

Yes, It Works

The Sony HVR-MRC1 can record DV from the Panasonic DVX100 via FireWire. Multiple users on DVXuser have confirmed this exact combination works. One called it "the best tapeless solution for a DVX." The MRC1 records the DV stream onto CompactFlash cards, giving you instant .dv or .avi files ready for editing — no tape capture needed.

Why Yours Isn't Working

The most likely causes, in order of probability:

  1. CAM LINK needs to be OFF. With a non-Sony camera, the MRC1 can't sync to the camera's record button. You must disable CAM LINK and use the MRC1's own start/stop button.
  2. The DVX100 isn't outputting DV over FireWire. You need AV IN/OUT SETUP → DV OUT → ON in the camera menu, and the camera must be in Camera mode (not VCR mode) with a tape loaded, or actively recording.
  3. The MRC1's i.LINK MODE is set wrong. It must be set to AUTO or DV (not HDV).
  4. The MRC1 isn't powered. When used via the i.LINK cradle (not mounted on a Sony camera), the MRC1 needs its own power: an L-series battery or AC adapter.
  5. The CompactFlash card isn't formatted. The MRC1 requires CF cards formatted by the unit itself.

Do You Have the Cradle?

The MRC1 alone has no i.LINK port — it uses a proprietary shoe connector designed for the Z7/Z5/S270. To connect it to any other camera via FireWire, you need the HVRA-CR1 cradle. If you bought the MRC1K kit, it was included. If you bought just the MRC1 unit off eBay, you may be missing the cradle, and it won't work without it.

Step 1: DVX100 Camera Settings

The DVX100 must be actively outputting a DV stream over its FireWire port. Here's how to verify:

1

Enable DV OUT

Go to MENU → AV IN/OUT SETUP → DV OUT → ON. This enables the FireWire output. If this is set to OFF, the DVX100 will not send any DV signal over FireWire, even though the port is physically connected.

2

Camera Mode (Not VCR Mode)

For live recording, the DVX100 must be in Camera mode (the rear switch). In Camera mode with a tape loaded and the camera recording (or even just powered on), the DVX100 continuously outputs its DV signal over FireWire. When a tape is loaded, the camera is in FireWire output mode — it sends the signal out.

VCR mode is for playback and pass-through of existing tapes, not live camera output.

3

Load a Tape (Important)

Several users have reported that having a tape in the transport is necessary for the FireWire output to be active in Camera mode. The DVX100 outputs DV over FireWire while recording to tape — the FireWire stream is essentially a copy of what's going to tape. Without a tape, some DV cameras don't output a FireWire signal at all.

If you want to go fully tapeless (no tape in camera), test whether your specific DVX100 variant outputs over FireWire without a tape. Some do, some don't.

DVX100B Bonus Feature

The DVX100B specifically added a "synchro lock" feature on its IEEE 1394 terminal that allows it to remotely start/stop an external DV recording device. It supports three modes: record only to external, record to both tape and external simultaneously, or begin external recording when tape runs out. Check your DVX100B manual for EXT REC settings if you have the B model.

Step 2: HVR-MRC1 Menu Settings

The MRC1 has a small LCD screen and a few buttons for navigation (MENU, PREV/REW, NEXT/FF, PLAY/EXEC). Press MENU to enter the menu, use PREV/NEXT to navigate, and PLAY/EXEC to select.

Critical Settings

CAM LINK — Must Be OFF

There's a physical CAM LINK button on the top of the MRC1. Make sure it's disabled (the CAM LINK indicator should NOT be lit). With CAM LINK on, the MRC1 tries to sync its start/stop to the camera's record trigger, which only works with compatible Sony cameras.

Additionally, in the menu: SETTING → CAMLINK SEL — this determines the sync mode (SYNCHRO or FOLLOW). With a non-Sony camera, this setting doesn't matter because CAM LINK should be off entirely. But if you're experimenting, set it to FOLLOW (designed for older cameras).

i.LINK MODE — Set to AUTO or DV

Navigate to i.LINK MODE in the VIDEO mode menu. Options are:

  • AUTO (default) — Automatically detects HDV or DV signal. Should work.
  • DV — Forces DV-only mode. Try this if AUTO isn't detecting the signal.
  • HDV — HDV only. The DVX100 outputs DV, NOT HDV, so this will definitely not work.

Set this to AUTO first. If that doesn't work, explicitly set it to DV.

DV FILETYPE

Navigate to SETTING → DV FILETYPE. Options:

  • DV — Records .dv files. Works better with Mac/FCP.
  • AVI — Records .avi files. Works better with Windows/Premiere.

Since you're on a Mac, choose DV.

REC MODE — Set to NORMAL

Navigate to REC MODE. Set it to NORMAL. The other modes (CACHE, INTERVAL, LOOP) are specialty modes that could prevent recording if not configured correctly.

Step 3: Physical Connection & Power

FireWire Safety Warning

The DVX100 is notorious for blown FireWire ports. NEVER plug or unplug the FireWire cable while either device is powered on. Static discharge through the cable can fry the 1394 controller chip. Always: power off both devices → connect the cable → then power on. This is especially critical with 6-pin FireWire on Macs, but applies to all connections.

1

Power Off Everything

Turn off the DVX100 and the MRC1.

2

Insert MRC1 into Cradle

Slide the HVR-MRC1 into the HVRA-CR1 i.LINK cradle until it clicks. Attach an L-series battery to the cradle, or connect the AC adapter.

3

Insert CompactFlash Card

Insert a formatted CompactFlash card into the MRC1. If the card is new, you'll need to format it in the MRC1 first: FORMAT in the VIDEO mode menu.

4

Connect FireWire Cable

Connect the 4-pin FireWire cable from the DVX100's DV terminal to the i.LINK port on the MRC1 cradle. Make sure the cable is seated firmly and straight — these connectors are fragile.

5

Power On the MRC1 First

Press the POWER button on the MRC1. Wait for it to boot and show the status screen on its LCD.

6

Power On the DVX100

Turn on the DVX100 in Camera mode with a tape loaded. The MRC1 should detect the DV signal — you'll see the input format indicated on the MRC1's LCD screen.

Important: Disconnect Before Changing Format

From the Sony manual: "Before changing the HDV/DV format on the camcorder, disconnect the i.LINK cable. Changing the format after connecting the i.LINK cable may cause the video signal to not be recognized correctly." If you're having detection issues, power everything off, disconnect the cable, configure both devices separately, then reconnect.

Step 4: Recording Workflow

With a non-Sony camera, recording is a two-button operation. The MRC1 and DVX100 operate independently.

1

Start Recording on the DVX100

Press the record button on the DVX100. The camera starts recording to tape and simultaneously outputs the DV stream over FireWire.

2

Start Recording on the MRC1

Press the START/STOP button on the MRC1. The MRC1 begins capturing the incoming DV stream to CompactFlash. The LCD will show recording status, time, and remaining capacity.

3

Stop Recording

Press START/STOP on the MRC1 to stop its recording. Press the record button on the DVX100 to stop the camera. Order doesn't matter much, but stopping the MRC1 first ensures you don't lose the tail end of a clip.

You Can Leave the MRC1 Running

You don't have to start/stop the MRC1 for every take. You can press START/STOP on the MRC1 once and let it record continuously while you start and stop the DVX100 as usual. The MRC1 will record everything the DVX100 sends over FireWire into one long file (split at 4GB boundaries, about 20 minutes each, due to FAT32). Later, you can trim in your editor. This is simpler than trying to sync two record buttons on every take.

DVX100B Users: Automatic Sync

If you have the DVX100B (not the original DVX100 or 100A), it has an external recording control feature that may be able to trigger the MRC1 automatically over FireWire. Check your camera menu for EXT REC settings. This would eliminate the two-button problem.

Troubleshooting: Why It Won't Record

Work through these in order. Each one is a known cause of failure with this combination.

1. MRC1 Doesn't See Any Signal

  • Is the MRC1 in VIDEO mode? The MRC1 auto-selects VIDEO mode when connected to a camera and COMPUTER mode when connected to a PC. If the LCD shows COMPUTER mode, something is wrong with the connection. Power cycle both devices.
  • Is DV OUT enabled on the DVX100? Check MENU → AV IN/OUT SETUP → DV OUT → ON.
  • Is the DVX100 in Camera mode? Check the rear switch. VCR mode won't output a live camera signal.
  • Is there a tape in the DVX100? Some DV cameras don't output over FireWire without a tape loaded.
  • Is the MRC1's i.LINK MODE correct? Set it to AUTO or DV (not HDV).
  • Did you connect the cable before powering on? Try: power off both → connect cable → power on MRC1 → power on DVX100.
  • Is the FireWire cable good? You said it works fine DVX100-to-MacBook, so the cable is likely fine, but verify it's a 4-pin to 4-pin cable (the MRC1 cradle uses 4-pin).

2. MRC1 Sees Signal but Won't Record

  • Is CAM LINK off? This is the #1 cause. With CAM LINK on, the MRC1 waits for a record trigger from the camera. The DVX100 can't send this trigger (it's a Sony proprietary protocol). Turn CAM LINK OFF and use the MRC1's own START/STOP button.
  • Is the CompactFlash card formatted? A new or computer-formatted card won't work. Format it in the MRC1: FORMAT in the menu. Warning: some users reported that "FORMAT" killed certain CF cards. Try DELETE → ALL first if the card was previously used in the MRC1.
  • Is the CF card fast enough? Must be 133x speed or faster, 2GB or larger.
  • Is REC MODE set to NORMAL? CACHE, INTERVAL, or LOOP modes have specific trigger requirements that could prevent recording.

3. MRC1 Records but Files Are Corrupt/Empty

  • Don't open the CF slot during recording. This will stop recording and may corrupt the current file.
  • Don't lose power during recording. Use AC power when possible, or a fully charged L-series battery.
  • Check firmware version. Firmware V01.00 had bugs with certain SanDisk CF cards and with file imports into Final Cut Pro. Update to V02.00 (see below).
  • "Fatal error of originating software" — Power cycle the MRC1 (turn off, turn on).

4. Timecode Issues

With a non-Sony camera, the timecode on the MRC1's recordings won't match the timecode on the DVX100's tape. This is a known limitation. The MRC1 generates its own timecode. If you're recording to both tape and CF simultaneously and plan to sync them, be aware of this 3-4 frame offset.

Firmware Update

Check your MRC1's firmware version in the menu. If it shows Version:01, update to V02.00. This update fixes several real bugs:

  • Cache recording failures with SanDisk high-speed CF cards
  • Internal flash ROM data error causing unit malfunction
  • Multiple files imported as a single file in Final Cut Pro
  • Files recorded from HVR-1500 in DV/AVI mode couldn't be imported into FCP

Download from Sony's support page: https://pro.sony/en_GB/support-resources/software/mpengb00000129

Firmware Update Warning

Do NOT update firmware on battery power. Use the AC adapter only. If power is lost during the upgrade, the unit will need to be sent to an Authorised Sony Service Centre for repair. Insert the MRC1 into its cradle, connect AC power, then follow the update procedure.

File Formats & Importing

The MRC1 creates a folder structure on the CF card: VIDEO/HVR/ in the root directory. Files are stored there.

DV Signal (from DVX100)

DV FILETYPE Setting File Extension Best For
DV .dv Mac / Final Cut Pro
AVI .avi Windows / Premiere Pro

Files are split at 4GB boundaries (approximately 20 minutes of DV recording each) due to FAT32 limitations. Sony provides a utility to merge split files, but most NLEs handle them fine as individual clips.

Filename Format

Files are named: XX_CCCC_YYYY-MM-DD_hhmmss.dv (or .avi), where XX is the clip number, CCCC is the camera number, and the rest is a timestamp.

Importing to Mac

Pop the CF card into a card reader (FireWire 800 CF readers are fastest). In Final Cut Pro 7, use Log & Transfer — it recognizes MRC1 footage natively. Users report it's about 7x faster than real-time tape capture. In modern editors (Resolve, Premiere, FCPX), just import the .dv files directly.

DV vs DVCAM

The DVX100 records standard DV (25 Mbps). The MRC1 accepts DV, DVCAM, and HDV. There's no difference in recording time on CF between these formats — a 16GB card holds about 72 minutes regardless. DV and DVCAM both record continuously without the "clip restart" issue that HDV has.

Limitations with Non-Sony Cameras

The HVR-MRC1 was designed specifically for the Sony HVR-Z7U, HVR-Z5U, and HVR-S270U. When used with any other camera (including the DVX100), you lose several features:

  • No synced start/stop. You must manually start/stop recording on the MRC1 separately from the camera (unless you have a DVX100B with EXT REC control).
  • No viewfinder status. On Sony cameras, remaining time and recording status appear in the viewfinder. With the DVX100, you have to check the MRC1's own LCD.
  • No automatic power-on. On Sony cameras, mounting the MRC1 on the shoe connector powers it automatically. With the cradle, you manage power separately.
  • Timecode mismatch. The MRC1 generates its own timecode that won't match the camera's timecode.
  • SD only. The DVX100 outputs standard definition DV. The MRC1 can handle this, but you won't get any HDV features.

CF Card Formatting Tip

Some users reported that using "FORMAT" on certain CF cards in the MRC1 killed the cards permanently — they wouldn't reformat on any device afterward. If your card already has MRC1 data on it, use DELETE → ALL instead of FORMAT to clear it. Only use FORMAT on a brand-new card that's never been used in the MRC1. Also: firmware V02.00 fixes issues with SanDisk high-speed CF cards specifically.
